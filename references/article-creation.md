# Long-Form SEO Article Creation (Author)

Goal: Generate comprehensive 3,000+ word SEO-optimized articles with research, internal links, and structured data.

## Pre-Writing Checklist

Before writing, confirm:
- [ ] Primary keyword confirmed
- [ ] Search intent identified (informational/commercial/transactional)
- [ ] Top 3 competing articles reviewed (via web search)
- [ ] Word count target set (min 2,000, pillar pages 3,500+)
- [ ] Internal link targets identified (existing pages to link to)
- [ ] Target audience defined

## Article Structure Template

### 1. Title (H1)
- Include primary keyword near the start
- Add a power word (Ultimate, Complete, Best, Proven, Step-by-Step)
- Include the year if content is evergreen with updates
- Under 65 characters for SERP display
- Example: "Best Podcast Microphones for Beginners (2025 Guide)"

### 2. Meta Description
- 140-160 characters
- Include primary keyword
- Include a benefit or call-to-action
- Do NOT just repeat the title

### 3. Introduction (150-200 words)
- Hook: Open with the reader's pain point or goal
- Promise: Tell them exactly what they'll learn
- Keyword: Use primary keyword naturally in first 100 words
- NO fluff — get to value immediately

### 4. Table of Contents
- Link to all H2 sections
- Helps with jump links and signals structure to Google

### 5. Body Sections (H2s and H3s)
Structure depends on intent:

**For How-To / Tutorial content:**
- H2: What is [Topic] (brief definition, 200 words)
- H2: Why [Topic] Matters (benefits, 200 words)
- H2: Step 1 — [First Step] (300 words)
- H2: Step 2 — [Second Step] (300 words)
... (continue for all steps)
- H2: Common Mistakes to Avoid (200 words)
- H2: FAQ (5-7 questions, 300 words)

**For Listicle / Best-Of content:**
- H2: How We Evaluated [Items] (criteria, 200 words)
- H2: [Item 1] — [Brief Description]
  - H3: Key Features
  - H3: Pros and Cons
  - H3: Who It's Best For
... (repeat for each item)
- H2: Comparison Table
- H2: Final Verdict

**For Comparison content ([A] vs [B]):**
- H2: Quick Verdict (summary table)
- H2: Overview of [A]
- H2: Overview of [B]
- H2: Key Differences: [Feature by Feature]
- H2: Which Should You Choose?
- H2: FAQ

### 6. Data Visualizations & Tables
Include at least one of:
- Comparison table
- Feature matrix
- Step-by-step numbered list
- Stat callout box (use real data from web research)

### 7. Internal Links
- Add 3-5 internal links to existing related content
- Use descriptive anchor text containing keywords
- Link naturally within sentences, not in a list at the bottom
- Format: "Learn more in our guide to [topic]" or "as we cover in [article title]"

### 8. External Links
- Link to 1-2 authoritative external sources (studies, official docs)
- Opens in new tab (note this in CMS)
- Use for statistics, definitions, official resources

### 9. FAQ Section
- 5-7 questions that match "People Also Ask" results
- Each answer 40-100 words
- Questions should include secondary keywords
- Use H3 for each question (enables FAQ schema)

### 10. Conclusion (150-200 words)
- Summarize key takeaways
- Repeat primary keyword naturally
- Include a CTA (next article to read, product to try, newsletter)

## Writing Quality Standards

**Readability:**
- Sentences under 25 words average
- Paragraphs 2-4 sentences max
- Use bullet points and numbered lists for any list of 3+ items
- Bold key terms on first use
- Use subheadings every 300 words

**Voice:**
- Write for humans first, search engines second
- Use second person ("you") to address the reader directly
- Avoid passive voice where possible
- Be specific — use numbers, examples, and named tools/products

**E-E-A-T Signals (Google's quality framework):**
- Cite experience: "In our testing...", "Based on our analysis..."
- Add expertise: Include data, expert quotes, or case studies
- Show authority: Reference credible external sources
- Build trust: Be honest about limitations, pros AND cons

---

## SEO Optimization Pass (run after writing)

After generating the article, check:
1. Keyword density: Primary keyword appears 1-2% of total words (use naturally)
2. LSI keywords: Include 5-10 related terms (use web search to find "related searches")
3. Heading structure: H1 > H2 > H3 hierarchy is logical
4. Image alt text: Every image has a descriptive alt tag with keyword where natural
5. URL slug: Short, keyword-rich, hyphenated (e.g., /best-podcast-microphones/)

## Schema Markup Recommendations

Add this schema to the page based on content type:
- How-To articles: HowTo schema
- FAQ sections: FAQPage schema (always add if you have a FAQ section)
- Product reviews: Review / AggregateRating schema
- News/current events: NewsArticle schema
- General articles: Article schema with author and datePublished
